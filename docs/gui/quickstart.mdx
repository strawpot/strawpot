## Prerequisites

- StrawPot installed (`pip install strawpot`)
- At least one agent runtime set up (e.g. Claude Code)

## Launch the Dashboard

```bash
strawpot gui
```

This starts the web server on [http://localhost:8741](http://localhost:8741). Use `--port` to change:

```bash
strawpot gui --port 9000
```

## Register a Project

1. Go to **Projects** in the sidebar
2. Click **Add Project**
3. Browse to a directory on your machine
4. The project name is auto-detected from the directory name
5. Choose an **Isolation** strategy (`none` or `worktree`) and **Merge Strategy** (`auto`, `local`, or `pr`) — these default to your global config values
6. Click **Create**

If you select `worktree` isolation and the directory is not a git repository, you'll be prompted to initialize one with `git init`.

The dashboard scans the project's `.strawpot/sessions/` directory to import any existing session history.

## Launch a Session

1. Open a project from the project list
2. Click **Launch Session**
3. Fill in the launch dialog:
   - **Task** — what the agent should do (required)
   - **Context Files** — attach project files for the agent to reference
   - **Role** — orchestrator role (defaults to project config)
4. Expand **Advanced Options** for more controls:
   - **Interactive mode** — allow the agent to ask you questions via a chat panel
   - **Runtime** — agent runtime override
   - **Memory** — memory provider override
   - **System Prompt** — additional instructions appended to the role prompt
   - **Max Delegations** — cap delegation calls per session
   - **Cache** settings — delegation cache controls
5. Click **Launch**

The session runs as a headless subprocess. You can monitor progress from the
session detail page, which shows the agent delegation tree, trace events, and
live log output.

## Start a Conversation

Conversations let you have multi-turn interactions with agents. Each message you send creates a new session; StrawPot automatically prepends a summary of prior turns so the agent has full context.

1. Open a project and go to the **Conversations** tab
2. Click **New Conversation**
3. Type your first message and press **Send** (or Cmd/Ctrl+Enter)
4. The agent runs as a session — you can click **View session** to inspect the delegation tree and trace
5. Once complete, type a follow-up and the agent will pick up where it left off

Conversations are useful for iterative tasks: reviewing code in stages, asking follow-up questions, or breaking a large task into sequential steps.

### How context handover works

Each new session in a conversation receives a **Prior Conversation** prefix containing:

- **History** — A condensed summary of each prior turn (what was asked, what was done). Recent turns retain more detail than older ones. Only the most recent 10 turns are included.
- **File changes** — Which files were modified in recent turns, so the agent knows where to look.
- **Pending Follow-up** — For the last turn, includes both the structured recap (up to 1500 chars) and a tail of the raw agent output (last 1500 chars). The recap tells the next agent *what to know*; the raw output shows *what actually happened*.
- **Conversation history file** — Full session outputs are written to `.strawpot/conversation_{id}_history.md` in the project directory before each session (one file per conversation). The agent can read this file on demand for more detail than the condensed summaries provide. Recent turns (last 5) include full output; older turns include recap only.
- **Recap instruction** — Asks the agent to end its response with a structured `## Session Recap` section covering what was accomplished, files changed, key decisions, and open items. The recap is extracted for use in subsequent turn summaries.

This is separate from [memory](/concepts/memory), which operates across conversations. Both work together: conversation context provides the sequential narrative, while memory provides long-lived knowledge.

## Browse Resources

The **Resources** section in the sidebar lists installed packages by type:

- **Roles** — orchestrator and sub-agent role definitions
- **Skills** — reusable skill modules
- **Agents** — agent runtime wrappers
- **Memory** — memory providers

Click any resource to view its details. Use **Install Resource** on a project
page to install resources from StrawHub directly.

## Project Resources

Each project page has a **Resources** tab showing which roles, skills, and
agents are installed locally in that project. From here you can:

- Install new resources from StrawHub
- Update existing resources to the latest version
- View resource details and dependencies

## Upload Context Files

On the project's **Files** tab, drag and drop files to upload them as context
for future sessions. Uploaded files are stored in the project's
`.strawpot/files/` directory and can be attached when launching a session.

## Configuration

The **Settings** page lets you edit the global `~/.strawpot/strawpot.toml`.
Each project also has a config editor accessible from the project detail page.

Changes are written directly to the TOML files — the same files the CLI reads.
