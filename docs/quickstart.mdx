## Prerequisites

- Python 3.11+
- Git
- npm (for installing agent CLIs) or an AI agent CLI already installed
- tmux (recommended, for interactive sessions)

## Install

<CodeGroup>
```bash pip
pip install strawpot
```

```bash From source
cd cli
pip install -e ".[dev]"
```
</CodeGroup>

## Start a Session

Run `strawpot start` from any git repository:

```bash
cd your-project
strawpot start
```

### First run

On first run, StrawPot detects that no agent is configured and launches an
interactive **onboarding wizard** that walks you through:

1. **Agent selection** — choose from Claude Code, Gemini CLI, Codex, OpenHands, or Pi
2. **Agent installation** — installs the wrapper and underlying CLI tool automatically
3. **Configuration** — saves your chosen runtime to `~/.strawpot/strawpot.toml`

After onboarding completes, the session starts normally.

<Note>
If you run `strawpot start --headless` before completing onboarding, the command
exits with an error listing the manual configuration steps.
</Note>

### Normal startup

On subsequent runs, `strawpot start`:
1. Loads configuration from `strawpot.toml` (if present) and `~/.strawpot/strawpot.toml`
2. Resolves the agent runtime (default: Claude Code)
3. Starts a Denden gRPC server on `127.0.0.1:9700`
4. Spawns the orchestrator agent in your terminal

### With Options

```bash
# Use a specific orchestrator role
strawpot start --role team-lead

# Use a specific agent runtime
strawpot start --runtime strawpot-claude-code

# Non-interactive mode with a task
strawpot start --task "Fix the failing tests in src/auth.py"
```

## Install Skills and Roles

StrawPot uses [StrawHub](https://strawhub.dev) for reusable skills and roles:

```bash
# Install a skill
strawpot install skill git-workflow

# Install a role (with dependency resolution)
strawpot install role implementer

# Search the registry
strawpot search "code review"
```

## Configuration

Create a project-level config file:

```toml
# strawpot.toml (project root)
runtime = "strawpot-claude-code"
isolation = "worktree"

[orchestrator]
role = "team-lead"

[policy]
max_depth = 3

[agents."strawpot-claude-code"]
model = "claude-sonnet-4-6"

[memory]
provider = "dial"  # default; set to "" to disable
```

See [Configuration](/cli/configuration) for the full reference.

## What Happens During a Session

When the orchestrator agent needs help, it delegates sub-tasks via Denden:

1. **Policy check** — Is the requested role allowed? Is the depth limit reached?
2. **Role resolution** — StrawPot fetches the role and its dependencies from StrawHub
3. **Memory retrieval** — The memory provider supplies context from past sessions
4. **Agent spawn** — A sub-agent is launched with the role's instructions, skills, and memory context
5. **Result** — The sub-agent's output is recorded to memory and returned to the caller

All agents work in the same directory (worktree or project dir), so they can see each other's changes.

## Web Dashboard

Prefer a browser-based interface? The GUI is included with StrawPot:

```bash
strawpot gui
```

The dashboard lets you manage projects, launch sessions, browse resources, and
set up scheduled tasks — all from [http://localhost:8741](http://localhost:8741).
See the [Web Dashboard](/gui/index) guide for details.
