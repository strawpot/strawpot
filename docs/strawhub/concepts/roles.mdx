---
title: Roles
description: Agent behavior definitions with dependencies and agent config
---

A role defines agent behavior, default agent runtime, and task specialization. Roles depend on skills (for instructions) and optionally on other roles (for delegation).

## Format

Every role is a directory containing a `ROLE.md` file:

```yaml
---
name: implementer
description: "Writes code to implement features and fix bugs"
metadata:
  strawpot:
    dependencies:
      skills:
        - git-workflow
        - code-review
        - python-testing
      roles:
        - reviewer
    default_agent: claude_code
---

# Implementer

Role instructions for the agent...
```

### Required Fields

| Field | Description |
|-------|-------------|
| `name` | Package slug (unique identifier) |
| `description` | One-line summary |

### Optional Fields

| Field | Description |
|-------|-------------|
| `metadata.strawpot.dependencies.skills` | List of skill dependencies |
| `metadata.strawpot.dependencies.roles` | List of role dependencies |
| `metadata.strawpot.default_agent` | Default agent runtime name |

## Dependencies

Roles use a structured object with `skills` and `roles` sub-keys:

```yaml
metadata:
  strawpot:
    dependencies:
      skills:
        - git-workflow
        - code-review
      roles:
        - reviewer
```

- **Skill dependencies** provide instructions the agent follows
- **Role dependencies** define other roles the agent can delegate to

## Default Agent

Roles can specify a default agent runtime:

```yaml
metadata:
  strawpot:
    default_agent: claude_code
```

This can be overridden by the user's `runtime` config in their StrawPot configuration.

## How Roles Are Used

When StrawPot receives a delegation request:

1. **Resolve** тАФ Fetch the role from StrawHub with all transitive dependencies
2. **Stage** тАФ Place role and skill files in the agent's workspace
3. **Build prompt** тАФ The `ROLE.md` body becomes the system prompt, with skill instructions appended
4. **Spawn** тАФ Launch a sub-agent with the assembled context

The agent sees:

```
<agent-workspace>/
тФЬтФАтФА skills/
тФВ   тФЬтФАтФА git-workflow/SKILL.md
тФВ   тФФтФАтФА code-review/SKILL.md
тФФтФАтФА roles/
    тФФтФАтФА implementer/ROLE.md
```

## Roles vs Skills

| | Skills | Roles |
|---|--------|-------|
| **Purpose** | Instructions and workflows | Agent behavior definition |
| **Can depend on** | Other skills | Skills and other roles |
| **Default agent** | No | Yes |
| **Used by** | Loaded into agent context | Assigned to agents during delegation |
| **Namespace** | Separate from roles | Separate from skills |
