Skills, roles, and agents use YAML frontmatter in their markdown files. This page documents the complete schema.

## Skill Frontmatter (`SKILL.md`)

```yaml
---
name: code-review                          # Required: package slug
description: "Code review checklist"       # Required: one-line summary
metadata:
  strawpot:
    dependencies:                          # Optional: skill dependencies
      - security-baseline
      - git-workflow
      - python-testing
      - code-style
    tools:                                 # Optional: system tool requirements
      gh:
        description: GitHub CLI
        install:
          macos: brew install gh
          linux: apt install gh
          windows: winget install GitHub.cli
    env:                                   # Optional: required environment variables
      GITHUB_TOKEN:
        required: true
        description: GitHub API token
---

# Skill Title

Markdown body with instructions for the agent...
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Package slug (unique within skills) |
| `description` | Yes | One-line summary |
| `metadata.strawpot.dependencies` | No | Flat list of skill dependency slugs |
| `metadata.strawpot.tools` | No | System tool requirements |
| `metadata.strawpot.env` | No | Required environment variables (prompted at session start) |

## Role Frontmatter (`ROLE.md`)

```yaml
---
name: implementer                           # Required: package slug
description: "Implements features"          # Required: one-line summary
metadata:
  strawpot:
    dependencies:                           # Optional: dependencies
      skills:                               #   skill dependencies
        - git-workflow
        - code-review
      roles:                                #   role dependencies
        - reviewer
    default_agent: claude_code               # Optional: default agent runtime
---

# Role Title

Markdown body with role instructions for the agent...
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Package slug (unique within roles) |
| `description` | Yes | One-line summary |
| `metadata.strawpot.dependencies.skills` | No | List of skill dependencies |
| `metadata.strawpot.dependencies.roles` | No | List of role dependencies (use `"*"` for all available roles) |
| `metadata.strawpot.default_agent` | No | Default agent runtime name |

## System Tools Schema

```yaml
metadata:
  strawpot:
    tools:
      <tool-name>:
        description: <string>          # What the tool is
        install:
          macos: <shell command>       # macOS install command
          linux: <shell command>       # Linux install command
          windows: <shell command>     # Windows install command
```

- Tool names should match the CLI binary name (used for `PATH` detection)
- Install commands run in the user's shell
- Only the current OS command is executed
- Supported OS keys: `macos`, `linux`, `windows`

## Environment Variables Schema

```yaml
metadata:
  strawpot:
    env:
      <VARIABLE_NAME>:
        required: <boolean>          # Whether the variable must be set
        description: <string>        # Human-readable description
```

- Resolution order: `os.environ` > saved config (`[skills.<slug>.env]` in `strawpot.toml`) > interactive prompt
- Prompted values are automatically saved to the project-local `strawpot.toml` for future sessions
- During delegation, missing required variables cause the delegation to fail (saved config values are checked)
- Variables set at session start are inherited by all sub-agents
- If the same variable appears in multiple skills, `required: true` takes precedence

## Agent Frontmatter (`AGENT.md`)

```yaml
---
name: my-agent                              # Required: agent slug
description: "My custom agent"              # Required: one-line summary
metadata:
  version: "1.0.0"                          # Optional: semver version
  strawpot:
    bin:                                     # Required: compiled agent binary
      macos: my_agent_wrapper               #   keyed by OS
      linux: my_agent_wrapper
    tools:                                   # Optional: system tool requirements
      my-cli:
        description: My agent CLI
        install:
          macos: brew install my-cli
          linux: apt install my-cli
    params:                                  # Optional: custom parameters for the agent
      model:
        type: string
        default: gpt-4
        description: Model to use
    env:                                     # Optional: required environment variables
      MY_API_KEY:
        required: true
        description: API key for the service
---

# Agent Title

Markdown body describing the agent (shown to LLMs for discovery).
```

### Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Agent slug (unique identifier) |
| `description` | Yes | One-line summary |
| `metadata.version` | No | Semver version string |
| `metadata.strawpot.bin.<os>` | Yes | Compiled agent binary name, keyed by OS (`macos`, `linux`) |
| `metadata.strawpot.tools` | No | System tool requirements |
| `metadata.strawpot.params` | No | Custom parameters for the agent binary |
| `metadata.strawpot.env` | No | Required environment variables |

## Parameters Schema

```yaml
metadata:
  strawpot:
    params:
      <param-name>:
        type: <string>              # Parameter type
        default: <value>            # Default value
        description: <string>       # Human-readable description
```

- Defaults from `AGENT.md` are merged with user config from `[agents.<name>]` in `strawpot.toml`
- User values take precedence over defaults
- The merged config is passed to the agent binary as the `--config` JSON string

## Key Differences: Skills vs Roles vs Agents

| Aspect | Skills | Roles | Agents |
|--------|--------|-------|--------|
| File | `SKILL.md` | `ROLE.md` | `AGENT.md` |
| Dependencies | Flat list (skills only) | Structured (`skills` + `roles`) | System tools only |
| Default agent | No | `default_agent` field | N/A (is the agent) |
| Env vars | Yes | No | Yes |
| Params | No | No | Yes |
| Namespace | Separate | Separate | Separate |
