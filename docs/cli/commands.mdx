## Session Management

### `start`

Start an orchestration session.

```bash
strawpot start [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `--role ROLE` | Orchestrator role slug | From config or `orchestrator` |
| `--runtime RUNTIME` | Agent runtime name | From config or `strawpot-claude-code` |
| `--task TEXT` | Task for non-interactive mode (empty = interactive) | — |
| `--headless` | Run detached with output to log file (requires `--task`). Exits with error if not configured. | `false` |
| `--system-prompt TEXT` | Custom system prompt appended to role instructions | — |
| `--run-id ID` | Pre-assigned run ID (used by the GUI) | Auto-generated |
| `--pull MODE` | Pull before session: `prompt`, `auto`, `always`, `never` | From config or `prompt` |
| `--host HOST` | Denden gRPC host | From config |
| `--port PORT` | Denden gRPC port | From config |
| `--no-cache-delegations` | Disable delegation caching for this session | `false` |
| `--cache-max-entries N` | Max cached delegation results (0 = unlimited) | From config |
| `--cache-ttl-seconds N` | Max age for cached results in seconds (0 = unlimited) | From config |
| `--memory PROVIDER` | Memory provider override | From config |
| `--max-num-delegations N` | Max delegation calls per session (0 = unlimited) | From config |
| `--memory-task TEXT` | Clean task string passed to memory scoring (defaults to `--task`) | — |

**Examples:**

```bash
# Interactive session with defaults
strawpot start

# Custom orchestrator role
strawpot start --role team-lead

# Non-interactive task
strawpot start --task "Refactor the auth module"

# Headless session with custom system prompt
strawpot start --headless --task "Review PRs" --system-prompt "Focus on security issues"
```

<Note>
When run with `--headless`, StrawPot requires a configured environment (agent installed, runtime set). If first-run setup has not been completed, the command exits with an error listing the manual configuration steps. Run `strawpot start` interactively first to complete the onboarding wizard.
</Note>

### `gui`

Launch the web dashboard.

```bash
strawpot gui [--port PORT]
```

| Option | Description | Default |
|--------|-------------|---------|
| `--port PORT` | Port for the GUI server | `8741` |

Opens a browser-based dashboard for managing projects, launching sessions,
and scheduling automated tasks. See the [Web Dashboard](/gui/index) guide.

### `config`

Show the merged configuration (defaults + global + project + CLI).

```bash
strawpot config
```

### `sessions`

List all running sessions.

```bash
strawpot sessions
```

Displays session ID, status, uptime, working directory, and Denden address for each active session.

### `agents`

List agents in a specific session.

```bash
strawpot agents <session_id>
```

Shows each agent's role, runtime, parent agent, and status.

## StrawHub Passthrough

StrawPot delegates these commands to the `strawhub` CLI. They work identically to calling `strawhub` directly.

### `install`

Install a skill, role, or memory provider from StrawHub.

```bash
strawpot install skill <slug> [options]
strawpot install role <slug> [options]
strawpot install agent <slug> [options]
strawpot install memory <slug> [options]
```

| Option | Description |
|--------|-------------|
| `--global` | Install to global directory (`~/.strawpot`) |
| `--version VER` | Install a specific version |
| `--force` | Force replace existing (requires `--version`) |
| `--update` | Update to latest if already installed |
| `--skip-tools` | Skip system tool installation |
| `--yes` | Auto-confirm prompts |

### `search`

Search the StrawHub registry.

```bash
strawpot search <query> [--kind skill|role|memory|all] [--limit N]
```

### `list`

List available packages.

```bash
strawpot list [--kind skills|roles|memories|all] [--sort updated|downloads|stars]
```

### `info`

Show detailed information about a package.

```bash
strawpot info skill <slug>
strawpot info role <slug>
strawpot info memory <slug>
```

### `resolve`

Resolve a package to its installed path and dependencies.

```bash
strawpot resolve skill <slug>
strawpot resolve role <slug>
strawpot resolve memory <slug>
```

### `uninstall`

Remove an installed skill, role, agent, or memory provider.

```bash
strawpot uninstall skill <slug>
strawpot uninstall role <slug>
strawpot uninstall agent <slug>
strawpot uninstall memory <slug>
```

### `update`

Update installed packages to the latest versions.

```bash
strawpot update [--kind skill|role|agent|memory|all]
```

### `init`

Create a `strawpot.toml` config file from installed packages.

```bash
strawpot init
```

Scans installed roles, skills, and agents and generates a project config file.

### `install-tools`

Install system tools declared by installed packages.

```bash
strawpot install-tools
```

### `publish`

Publish a skill, role, or memory provider to StrawHub.

```bash
strawpot publish skill [<path>]
strawpot publish role [<path>]
strawpot publish memory [<path>]
```

### Authentication

```bash
strawpot login     # Authenticate with StrawHub
strawpot logout    # Remove stored credentials
strawpot whoami    # Show current user
```
